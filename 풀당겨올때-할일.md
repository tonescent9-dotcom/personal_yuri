# 📌 풀(pull) 당겨올 때 할 일들

> 이 저장소에는 **소스 코드만** 올라가 있어요.
> `node_modules/`, `dist/` 같은 건 각자 컴퓨터에서 명령어로 다시 만들어야 합니다.
> (`.gitignore`는 "올리지 마" 목록일 뿐, 파일을 자동으로 만들어주지는 않아요.)

---

## 상황 1. 이 저장소를 **처음** 받을 때 (새 컴퓨터 등)

```bash
git clone https://github.com/tonescent9-dotcom/personal_yuri.git
cd personal_yuri
npm install        # 라이브러리(node_modules) 설치 — 꼭 해야 함
npm run dev        # 개발 서버 실행
```

- 브라우저에서 뜨는 주소(보통 http://localhost:5173/) 로 접속하면 됩니다.

---

## 상황 2. 이미 받아둔 저장소에서 **최신 코드 당겨올** 때

```bash
git pull            # 최신 소스 받기
npm install         # ⚠️ package.json 이 바뀌었을 때만 (아래 설명)
npm run dev         # 개발 서버 실행
```

### `npm install` 은 언제 해야 하나요?
- `git pull` 결과에 **`package.json` / `package-lock.json` 변경**이 있으면 → `npm install` 한 번 실행
  (라이브러리가 추가/변경됐다는 뜻)
- 변경이 없으면 → **생략 가능**. 그냥 `npm run dev` 만 하면 됩니다.
- 헷갈리면 그냥 `npm install` 한 번 더 해도 아무 문제 없어요. (없으면 설치, 있으면 그대로 둠)

---

## 자주 쓰는 명령어

| 명령어 | 하는 일 |
|---|---|
| `npm run dev` | 개발 서버 실행 (코드 저장하면 화면 바로 반영) |
| `npm run build` | 배포용 결과물(`dist/`) 생성 |
| `npm run preview` | 빌드된 결과물 미리보기 |

### 📱 휴대폰(같은 Wi-Fi)에서 확인하고 싶을 때
```bash
npm run dev -- --host
```
- 실행하면 `Network: http://192.168.x.x:5173/` 같은 주소가 뜹니다.
- 폰을 **PC와 같은 Wi-Fi**에 연결하고 그 주소로 접속하세요.
- 안 열리면 대부분 **Windows 방화벽**이 원인 → Node.js에 "개인 네트워크" 허용.

## 커밋 & 푸쉬 (내 변경사항 올릴 때)

```bash
git add -A
git commit -m "무엇을 바꿨는지 한 줄 설명"
git push
```

- `node_modules/`, `dist/` 는 `.gitignore` 덕분에 자동으로 제외됩니다.
  (실수로 올라갈 걱정 X)

---

## 문제가 생기면 (초기화 방법)

무언가 꼬였을 때, node_modules 를 지우고 새로 설치하면 대부분 해결됩니다.

```bash
# PowerShell
Remove-Item -Recurse -Force node_modules
npm install
```

- `node_modules` 를 지워도 **코드에는 전혀 영향 없어요** (다시 설치하면 그대로 복구됨).
